# Hotkey-Driven Task & Memo Application

![Python](https://img.shields.io/badge/Python-3.x-blue.svg)
![Tkinter](https://img.shields.io/badge/UI-Tkinter-orange.svg)

## 概要 (Introduction)

このリポジトリは、バックグラウンドで常駐し、システムワイドなホットキーやタスクトレイから各種機能を呼び出すPython製デスクトップアプリケーションです。メモの記録、チケット管理、データ閲覧、ガントチャート生成などの機能を、作業を中断することなく即座に起動できます。

## 開発背景 (Context)

- QA業務中、バグ起票の度にDifyを開く手間
- テスト中に気になった仕様や挙動をその場でメモしたい
- タスクの登録と、タスクのボトルネックを記録

作業の文脈を切らずに上記を行うためにホットキーで呼び出せるツールを作りました。

## アーキテクチャ / 処理フロー (Architecture)

本アプリケーションは、イベントキューを中心としたアーキテクチャを採用しています。ホットキーリスナーやタスクトレイからのユーザー操作をイベントとしてキューに追加し、メインのUIスレッドが定期的にキューをポーリングして各処理を実行します。これにより、スレッドセーフな非同期処理を実現しています。

```text
+----------------------+      +------------------+      +--------------------+      +--------------------+
|   User Input         |----->|   Event Queue    |----->|   Event Processor  |----->|     UI Forms       |
| (Hotkey / Tray Menu) |      |  (queue.Queue)   |      | (in main UI loop)  |      | (Tkinter Windows)  |
+----------------------+      +------------------+      +--------------------+      +----------+---------+
                                                                                               |
                                                                                               | (on_submit)
                                                                                               v
                                                                                      +--------------------+
                                                                                      | Database Module    |
                                                                                      |   (save_entry)     |
                                                                                      +--------------------+
```

## 主な機能 (Features)

- **ホットキーによる機能起動:** システムのどこからでも、定義されたホットキーで各機能を呼び出します。
- **タスクトレイ常駐:** タスクトレイにアイコンを表示し、メニューからメモ入力フォームの表示やアプリケーションの終了が可能です。
- **メモ入力:** 簡易な入力フォームを表示し、入力されたテキストをデータベースに保存します (`save_entry`)。
- **データビューワー:** 保存されたデータを閲覧するためのビューワーを表示します。
- **チケット管理:** チケット作成用フォームの表示と、登録済みチケットの閲覧機能を提供します。
- **ガントチャート関連機能:** ガントチャート生成フォームや、プロジェクトのボトルネック分析用フォームを起動します。

## 使い方 (Usage)

本スクリプトをPythonインタプリタで直接実行することで、アプリケーションがバックグラウンドで起動し、タスクトレイにアイコンが表示されます。

```bash
python main.py
```

起動後は、設定されたホットキーを押すか、タスクトレイのアイコンをクリックすることで各機能を利用します。

**READMEに追記すべき情報:**
- 各機能 (`memo`, `viewer`, `ticket`など) に割り当てられている具体的なホットキーの組み合わせ。
- `db`モジュールが使用するデータベースの種類（例: SQLite）と、ファイルが生成されるパス。
- 外部ライブラリの依存関係をまとめた `requirements.txt` ファイル。

## 技術スタック (Tech Stack)

- **Python:** アプリケーションのコアロジックを実装する主要言語です。
- **Tkinter:** Pythonの標準GUIツールキット。クロスプラットフォームで動作する軽量なUIフォームを構築するために採用しています。
- **queue (Python標準ライブラリ):** スレッドセーフなキュー。ホットキーを監視する別スレッドからメインのUIスレッドへ、安全にイベントを伝達する役割を担います。

## 設計・実装のポイント (Key Points)

- **イベント駆動設計:** ユーザーからの入力をイベントとして抽象化し、`queue.Queue`で一元管理しています。これにより、UIの応答性を損なうことなく、外部からのトリガー（ホットキー）を処理できます。
- **UIの分離:** メインの`Tk`ルートウィンドウは非表示(`withdraw()`)とし、機能ごとに独立したトップレベルウィンドウを生成します。これにより、常駐型アプリケーションとしての自然な挙動を実現しています。
- **モジュール化による関心の分離:** `db` (データベース操作)、`ui` (UIコンポーネント)、`hotkey` (ホットキー管理) のように機能ごとにモジュールを分割し、コードの保守性と再利用性を高めています。
- **コールバックの活用:** UIからの入力確定処理 (`on_submit`) や、ホットキー・トレイからのイベント通知はコールバック関数を渡すことで実装されており、疎結合なコンポーネント設計となっています。
