# タスクログ機能設計

## 機能概要

タスクの進捗とボトルネックを記録し、引継ぎの容易化・得手不得手の可視化を目的とした機能。

---

## データ設計

### tasks テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー |
| name | TEXT | タスク名 |
| start_date | TEXT | 開始日（M/D） |
| end_date | TEXT | 終了予定日（M/D） |
| status | TEXT | 未着手 / 進行中 / 完了 / 引継ぎ済 |
| is_archived | INTEGER | 0: 現行 / 1: アーカイブ済 |

### bottlenecks テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| id | INTEGER | 主キー |
| task_id | INTEGER | 紐付くタスクID（tasks.id） |
| content | TEXT | ボトルネック内容 |
| created_at | TEXT | 記録日時（M/D HH:MM） |

---

## ステータスとSlack送信・アーカイブ

| ステータス | Slack送信 | アーカイブ |
|---|---|---|
| 未着手 | なし | なし |
| 進行中 | なし | なし |
| 完了 | タスク単体を送信 | なし |
| 引継ぎ済 | 全タスクまとめを送信 | される |

- ステータスを「引継ぎ済」に変えたタイミングで手動トリガー
- アーカイブされたタスクはis_archived=1で残す（削除しない）

---

## Slack出力フォーマット

**完了時（タスク単体）**
```
【仕様確認】3/31 - 4/2  完了
  ⚠ 3/31 仕様書が古くて確認に時間がかかった
  ⚠ 4/1  担当者が不在で確認待ちになった
```

**引継ぎ済時（全タスクまとめ）**
```
【仕様確認】3/31 - 4/2  完了
  ⚠ 3/31 仕様書が古くて確認に時間がかかった

【テスト実施】4/3 - 4/5  完了
  （ボトルネックなし）
```

---

## 操作フロー

1. ホットキーでタスク入力フォームを表示
2. タスク名・開始日・終了予定日・ステータスを入力してDBに保存
3. ホットキーでボトルネックフォームを表示
4. タスクを選択してボトルネック内容を入力（DBに保存のみ、Slack送信なし）
5. ステータスを「完了」に更新 → そのタスク単体をSlackに送信
6. ステータスを「引継ぎ済」に更新 → 全タスクまとめをSlackに送信＋アーカイブ
